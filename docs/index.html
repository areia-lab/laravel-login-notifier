Areia Lab — Laravel Login Notifier

Package: areia-lab/laravel-login-notifier — Notify users & admins when a login occurs from a new device or IP. Logs device, browser, platform, IP and optional geo-location.

Quick Install

Install the package (once published on Packagist or via VCS):

composer require areia-lab/laravel-login-notifier
# publish config + migration
php artisan vendor:publish --tag=login-notifier
php artisan migrate

Note: Geo-location support is optional. See Optional features below.

What it does

  • On Illuminate\Auth\Events\Login, captures IP, device, platform and browser using jenssegers/agent.
  • Optionally attempts geo-location when stevebauman/location is installed.
  • Stores a row in login_histories and sends an email to the user when the IP/device differs from the user's last login.
  • Optionally notifies an admin email address.

Configuration

After publishing, edit config/login-notifier.php:

return [
    'admin_email' => env('LOGIN_NOTIFIER_ADMIN_EMAIL', 'admin@example.com'),
    'notify_user' => true,
    'notify_admin' => true,
];

You can disable user or admin notifications by setting the flags to false.

Database Migration

The package ships with a migration that creates login_histories:

Schema::create('login_histories', function (Blueprint $table) {
    $table->id();
    $table->foreignId('user_id')->constrained()->cascadeOnDelete();
    $table->string('ip_address')->nullable();
    $table->string('device')->nullable();
    $table->string('platform')->nullable();
    $table->string('browser')->nullable();
    $table->string('geo_location')->nullable();
    $table->timestamps();
});

Listener behaviour

The listener attaches to the Login event and performs these steps:

  1. Collect IP and agent info.
  2. Attempt geo-location only if the stevebauman/location facade exists.
  3. Compare with the last login row for the user to determine whether this is a new device/IP.
  4. Persist the new login row and send notifications accordingly.

Safe geo-location (optional)

// inside the listener
$geo = null;
if (class_exists(\Stevebauman\Location\Facades\Location::class)) {
    try {
        $location = \Stevebauman\Location\Facades\Location::get($ip);
        $geo = $location ? $location->cityName . ', ' . $location->countryName : null;
    } catch (\Throwable $e) {
        $geo = null; // silently ignore errors
    }
}

Notifications

The package includes two mail notifications:

  • UserLoginAlert — informs the user about a new-device/IP login with IP, device and location.
  • AdminLoginAlert — sends a summary email to the admin email configured in login-notifier.php.

The notifications use Laravel's mail channel. Make sure mail is configured in your application (SMTP, Mailgun, etc.).

Optional packages

Package Purpose Install
jenssegers/agent Detect device/platform/browser composer require jenssegers/agent
stevebauman/location Resolve geo location from IP (optional) composer require stevebauman/location

If you want automatic installation of these dependencies, add them to your package composer.json require block. Otherwise the listener will work without geo-location.

Customization & Extensibility

  • Change detection logic: Modify the listener's comparison to include device strings, browser or a fingerprint instead of just IP.
  • Multiple notifications: Use additional channels (SMS, Slack) by implementing their notification channels.
  • Events: You can dispatch a custom event after creating LoginHistory for other apps to hook into. 
// Example: dispatch custom event
use App\Events\UserLoggedInDifferentDevice;

// after creating $loginHistory
event(new UserLoggedInDifferentDevice($user, $loginHistory));

Testing

Suggested tests:

  • Assert that a login from the same IP does not trigger a notification.
  • Assert that a login from a new IP triggers UserLoginAlert.
  • Mock the stevebauman/location responses when testing geo-location behavior. 
// Example PHPUnit-ish
public function test_new_ip_triggers_notification()
{
    Notification::fake();

    // create user & simulate login event with different IP
    // assert Notification::sent(UserLoginAlert::class)
}

Troubleshooting

  • Mail not sent: Verify MAIL_* env vars and queue configuration.
  • Geo-location null: Ensure stevebauman/location is installed and configured or accept null fallback.
  • Listener not firing: Ensure the service provider is registered (it is auto-discovered when installed via Composer) and that Event::listen(Login::class,...) executed (clear cache if needed).
